CgrtBilling

CgrtBilling (Carrier Grade Real Time Billing) is a billing system that charges for VoIP usage. CgrtBilling works integrated with at least 3 other big systems via JSON APIs and it has its own API to exchange information with others systems. CgrtBilling has many functionalities and it manages big amount of data. For example, there is a table that grows more than one million rows each month. For more details about the product, please check CgrtBilling.

I have been working in CgrtBilling for two and a half year. During the first year, I've been working alone with the founder of the company. After, the system grew up a lot and I invited a friend to help me. All the work have been done remote using Assembla to manage the use cases and SVN to control the versions.

Highlights

The main highlights that I consider during this project were:

  • Performance improvements. There was a routine of one of the legacy systems that spent 9 minutes to run. The same routine was reimplemented in CgrtBilling and, after applying some knowledge of data structures, the time has been reduced to less than one minute to run.
  • Replication. Master-Slave Postgres Replication and, in some cases, Master-Master Replication.
  • Dynamic filters screen. A screen where is possible to add filters dinamically, apply those filters to a set of objects. After apply the filter, run an action on selected objects. Take a look at dynamic filter screenshot and it will give an idea of how it works.
  • Long running tasks. Long run tasks are problematic in web applications. The user definitely will receive an timeout error if a task need 20 or 30 minutes to complete. In CgrtBilling, there are some tasks that need much time to run, then the user makes a request and a task is created and scheduled in a queue to run. He can monitor the status of his tasks while they are running.
  • Payment methods. The client can buy pre paid and pos paid plans using credit card and paypal as payment methods.

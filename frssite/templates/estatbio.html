EstatBio

In Brazil, there are 27 States and one TRE (Electoral Regional Tribunal) for each State. Each TRE is responsable for biometric gathering of its citzens.

EstatBio (Biometric Statistics) is a management system to monitor biometric gathering of citizens which reside in Ceara state. Biometric collections is a huge project to improve the security of Brazilian Elections and it is expected to finish in 2020.

Basically, EstatBio extracts summarized information from two huge Oracle databases and saves them in a small SQLite database and in JSON files. The ETL process occurs every dawn and once it is finished, Estatbio does not use Oracle production databases until the next update.

Highlights

  • Performance. The ETL process takes about 3 hours to finish, but once the summarized information is available, the system does not access production database anymore.
  • Integrity. In the past, all the information was in Spreadsheets and sometimes each manager had a different information. Now, the information is centralized, available for all the managers and there is an option to download data to spreadsheets also.
  • Visualization. The information is integrated with Google Maps API and Google charts API.
  • Summarized information. Estatbio is a software to help decision making, thus, it contains only the necessary information to make decisions.

Challenges

  • ETL. The ETL process could not run in during office hours and there was not a development database.
  • Distributed information and security policies. The ETL process had performance problems because the way that information is available and to respect security policies.
  • Managers opinion divergence. Managers have divergent option and sometimes is not easy to develop a system that satisfy all the parts.
